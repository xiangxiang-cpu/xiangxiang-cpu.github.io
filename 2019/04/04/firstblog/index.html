数据移动

  • 数据文件 ----- 数据库
  • 数据库 ----- 数据文件
  • 数据库 ----- 数据库

sqlload 用于外部数据文件–>数据库

  • control file（==重要==）：文本文件
  • input data file：数据源
  • bad file（如果有坏数据，则强行生成）：保存坏数据的文件（格式不对）【id本来应该是整型，但文件中是字符串】
  • discarded file（可选）：将格式正确，但不符合筛选条件的数据放在这里【只向数据库中插入工资大于2500的记录，不满足记录的放到这个文件中】
  • log file：（不同于重做日志文件）文本文件，sqlload出了问题可以查这个

sqlload数据类型

zoned和position适用于==固定长度的记录==

zoned：precision表示总的位数，scale表示小数点后的位数，源数据一个小数的小数点省略表示

position：源数据中字段间的分隔符省略

控制文件

数据较少时

数据较多，使用外部文件

  1. load data
  2. infile指定外部文件（数据源）
  3. 指定目标表
  4. 指定数据格式（记录、字段、字段名、类型、和Oracle不同）

  • 默认一行是一条记录

  • 如果两行一条数据，则append concatenate 2 into

  • 如果没规律，则要有标志表示新纪录的开始 continueif this (1:1)=’-’

    从第一列开始，到第一列结束

  • 字段的结束符 terminated by “,”

  • 可选定界符 optionally enclosed by ‘"’。char类型的如果不写这个，会将引号当作数据写入数据库

into table有多种模式

  • 默认为insert into，表里原来有数据的话，就不会覆盖，插不进去了
  • 表里已经有，则用append into
  • replace into，将表中的原来的数据删除后再插入（覆盖）

2.RMAN工具：Recover manage（恢复管理器，处理备份和恢复）
两种方式：冷备份（数据库未打开，mount状态下的备份，不是shutdown状态下）、热备份（数据库打开时做的备份）
1）冷备份：MOUNT
2）热备份：打开数据库的备份
认识RMAN：
进入RMAN：
F:\exp>rman
备份一个表空间users

lecture 14 RMAN下的整库备份和恢复

connect target sys/admin
backup tablespace users;//非归档模式只能做冷备份，不可做热备份
sys@db18c>shutdown immediate
sys@db18c>startup mount
sys@db18c>alter database archievelog;//修改数据库日志模式（非存档->存档模式）
