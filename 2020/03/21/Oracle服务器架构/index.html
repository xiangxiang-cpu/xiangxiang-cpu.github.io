Oracle服务器架构

服务器：提供服务的计算机，计算机经过优化处理，在某方面功能强大

文件服务器：存储容量强

应用服务器：CPU、内存强

Oracle服务器：数据存储、数据处理功能强大的计算机

==Oracle服务器 == instance + database==

  1. instance【播放器】【数据处理】【安装了Oracle软件的计算机】
  2. database【视频文件】【数据存储】

先装instance，再装database，否则会建库失败

两者不是绑定的，互相独立可以分离

集群

  • 设置多个冗余节点（一台Oracle服务器）
  • 节点之间有心跳线
  • 一台机器坏了，心不跳了，另一台就来接管

Oracle instance

  • 翻译：例程（例行程序，示例程序）
  • 访问数据库的方式
  • 总是打开一个且仅一个database

==实例 == 内存结构 + 一组后台进程==

内存结构

==内存 == 系统全局区域SGA + 程序全局区域PGA==

SGA

system global area，系统全局区域【分支机构都能访问的】

例程下可以有很多会话

  • 都在db12c同一个系统下，例程级别
  • instance启动时，根据初始化参数配置，向OS申请内存，所有会话可以共享
  • 不同数据库的内存不能共享
  • 一个用户select之后，文件留在内存（SGA），下一个用户读取就直接从内存获得内容

详细讲表 - 见Oracle知识点一栏

1 共享池

==共享池 == 库缓存 + 数据字典缓存==

库缓存：存放代码

（还能细分）代码放在这里（执行计划）（包括解析好的代码？？？、DLL）

每次解析SQL效率太低，在内存（共享池）中保存已经解析好的代码，避免重复解析

数据字典缓存：存放系统数据

数据分为

  • 系统数据（被Oracle维护和处理）== 数据字典 == 描述数据的数据 == 元数据 == 数据目录

    系统数据放在数据字典缓存中

  • 用户数据（被用户维护和处理的）

    用户数据放在数据库告诉缓存中

2 数据库高速缓存池：存放用户数据

存一些select的结果

【共享池】和【数据库高速缓存池】占90%空间

ps：其他的池都是可选的

  • java池，编译完了放在这里 - 通过java应用访问数据库时才需要
  • 通过c，没有c池。Oracle和微软对头，和sum好
3 重做日志缓存池

（重做）日志作用：恢复

出了问题，先恢复最近的备份，再用重做日志？？？

LGWR（log writer，后台进程之一）：将SGA中的重做日志缓存写入到磁盘中

在shutdown时，会把重做日志缓存池写磁盘，恢复见上面的例子

==日志，记录事务==，不是增删改等细化的操作

事务的ACID准则：原子性(Atomicity)、一致性(Consistency)、隔离性(Isolation)、持久性(Durability)

原子性

【转账，存钱】

要不都做，要不都不做

事务必须commit或者rollback，查询不是事务

  • 恢复redo，做对的东西再来一遍。【insert了100条数据，不知道什么原因没了，需要redo】
  • 撤销undo，把做错的东西撤销（对冲）掉

读的一致性

事务开始之前和结束之后，数据库要处于一致状态

别的数据库，事务未结束时，给表加锁，别的用户不能访问

eg 一个会话update，未commit；另一个会话访问的是旧数据

oracle：读一致性 – 修改时，数据先在高速缓存中；别的用户在回滚段中访问，不用等事务结束

当事务t1进行数据修改的时候，会将数据复制到回滚带上，如果另外的事务前来查询发现（高速缓存中的）数据被锁，会到回滚带上查询

隔离性

只能看到事务开始和结束的状态，中间状态是看不见的

用锁实现

持久性

持久 - 外存

临时 - 内存

commit之后，通知LGWR将重做日志缓存，写入重做日志文件中

重做日志缓冲区的内容

A. 对于每个dml或ddl语句，oracle服务器进程都会把用户内存空间的重做条目复制到重做日志缓冲区上

B. 重做条目包含：用于重做或重构dml或ddl操作对数据库所做更改的所需的信息，用于恢复数据库，并占据日志缓冲区的连续的有序的空间

lgwr进程

那缓冲区满了，该怎么办呢？这就需要lgwr进程来行动了

A. lgwr进程负责把重做日志缓冲区的内容写到磁盘上的活动的联机重做日志文件(或活动组的成员),他写入 自上次写入后的所有复制到缓冲区的条目

B. 重做日志缓冲区是一个循环的缓冲区，oracle服务器进程（非LGWR，LGWR是后台进程）负责把重做的条目覆盖重做日志缓冲区的已经写入磁盘的条目，lgwr的写入速度是非常快的，以确保重做日志缓冲区总有写入新条目的空间

触发LGWR开始工作的动作

A. 重做日志缓冲区的已使用的空间达到三分之一时
B. 当dbwn进程向磁盘写入已修改的缓冲区的时候
C. 每隔3秒钟
D. 用户提交事务处理时的一条提交记录(经常commit会及时刷新重做日志缓冲区空间)

PGA

program global area，程序全局区域

创建会话时才初始化

eg 排序区（order by子句）、游标在PGA中，会话之间不能共享

进程

  1. 用户进程
  2. 服务器进程
  3. 后台进程

用户进程

约等于UI，通过某个UI访问Oracle

sqlplus就是一个用户进程，sqldeveloper也是

用户进程不可以直接访问服务器进程【不能在银行柜台直接拿钱，要向柜台的人提请求】

服务器进程

==用于访问instance==

服务器进程是用户进程的代理，帮忙做事

eg

  • 用户执行一条update语句，看库缓存里有没有，如果做过产生很多计划，则直接调用执行；没有的话，重新执行语句？？？
  • 要改的数据在数据库高速缓存中有没有，有则直接修改；没有则从数据文件中调进来再改
  • update所在的事务还要被记到日志中
  • 用户要查找数据字典，在共享池的字典缓存里如果有，则直接访问；如果没有，则去数据文件中找？？？
专用服务器进程

【爸妈做饭】

专门为一个用户进程使用

效率高，支持用户进程数量少

适用：用户连接数 < 200

共享服务器进程

【饭店】

为多个用户进程共享

效率高，支持用户进程数量多

适用：200 - 3000个用户连接使用数

>3000需要配置中间件【食堂】

数据库文件

  • 内部文件 - 灰色，Oracle自己管理，都带SCN号
  • 外部文件 - 蓝色，也是Oracle文件，没有SCN号，但是性质等同于OS的文件，dba管理

控制文件，不能拷出来，再拷回去，这个文件是由Oracle处理的

这个文件是有checkpoint（SCN号）的，自带时钟，和时间有关系，拷出拷进会导致和其他文件的不一致
